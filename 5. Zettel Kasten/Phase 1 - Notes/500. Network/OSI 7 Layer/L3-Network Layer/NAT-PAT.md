
# NAT/PAT

## NAT(Network Address Translaction)

- 네트워크 주소를 변환하는 기술이다.
- 하나의 네트워크 주소에 다른 하나의 네트워크 주소로 변환하는 1:1변환이 기본
- [[IP]] 가 고갈되는 문제를 해결하기 위해 여러 개의 IP를 하나의 IP로 변환하기도 한다.
- 실제 공식 용어는 NAPT(Network Address Port Translation, RFC2663)
- 하지만 PAT(Port Address Translation) 을 실무에서 많이 사용한다.

## NAT/PAT의 용도와 필요성

### IP주소 고갈

- IPv4 주소 고갈문제로 사용됨
- 외부에 공개해야 하는 IP에 대해서는 공인 IP를 사용하고 외부에 공개할 필요가 없는 IP에 대해서는 사설 IP를 사용하여 효율적으로 IP를 사용하게 됨

### 보안

- IP주소는 네트워크 내에서 유일해야 한다. 따라서 외부와 통신할 때 사설 IP를 공인 IP로 변환하여 사용하면 외부에 사내 IP주소를 숨길 수 있다.

### 네트워크 통신

- 주소 체계가 같은 두 개의 네트워크를 통신 가능하게 해준다
    
    
- 회사 내에서는 사설 IP 주소가 중복이 되어도 상관없지만, 다른 네트워크와 통신할 때는 IP주소가 겹치면 충돌이 일어난다.
    
- 특히 회사간 통신(카드사, 은행)에서 이런 상황이 많이 발생한다.
    
- IP대역이 같은 네트워크와 통신할 가능성이 높은 네트워크를 연결하기 위해 출발지와 도착지를 한꺼번에 변환하는 “더블나트(Double NAT)” 기술을 사용한다.
    

### 불필요한 설정 변경을 줄인다

- KISA 를 통해 IP를 등록하고 사용하는 경우 IP는 본인 소유의 것이 아니라 임시로 빌려 쓰는 것이기 때문에 사업자를 이전하거나 IDC를 바꾸면 해당 공인 IP를 사용하는 설정들을 다 바꿔주어야 할텐데, NAT/PAT를 이용해서 네트워크를 구성했다면, 서버와 PC의 IP주소를 변경하지 않고 사업자 이전과 IDC 이전이 가능해 진다.

## NAT 동작 방식


📄 10.10.10.10이 20.20.20.20과 통신하는 과정

---

1. 웹 서버에 접근하기 위해 패킷에 출발지 10.10.10.10, 목적지 20.20.20.20 웹 서버 포트인 80포트로 채워 패킷을 보낸다.
2. NAT장비는 사용자가 보낸 패킷을 수신한 후 정책에 따라 통신가능한 공인 IP로 변경하고 NAT장비는 전후의 IP주소를 NAT테이블에 저장한다.
3. NAT장비에서 출발지 주소를 NAT장비에서 바뀐 11.11.11.11로 재작성하여 다시 보낸다.
4. 패킷을 받은 웹 서버는 응답을 주는데, 출발지 주소는 웹 서버의 주소인 20.20.20.20가 되고 목적지 IP는 NAT장비에서 변경된 공인 IP주소인 11.11.11.11이 된다.
5. 패킷을 받은 NAT장비는 목적지 주소에 대해서 11.11.11.11이 원래 10.10.10.10이라는 것을 확인한다.
6. NAT 변환 테이블에서 확인된 원래 패킷 출발지 10.10.10.10로 변경해 사용자에게 전송하면 최종적으로 사용자는 패킷을 수신한다. 

## PAT 동작 방식


1. 웹 서버에 접근하기 위해 패킷에 출발지 10.10.10.10, 목적지 20.20.20.20 웹 서버 포트인 80포트로 채워 패킷을 보낸다.
2. NAT장비는 외부 네트워크와 통신 가능한 11.11.11.11 으로 변환한다. 또한 다수의 사용자가 공인 IP를 써야 하기 때문에 출발지 포트도 변경하게 된다. 출발지 IP와 출발지 포트는 NAT장비에 의해 변경되고, 이 변경사항을 NAT 테이블에 기록한다.
3. NAT 장비에서 변경된 IP주소 11.11.11.11과 포트 3000으로 패킷을 재작성 해 다시 보낸다.
4. 요청을 받은 웹 서버는 응답을 보내게 되는데 출발지 는 웹 서버의 IP인 20.20.20.20이 되고 목적지는 NAT에 변환된 공인 IP 11.11.11.11가 3000포트가 적힌다.
5. 웹 서버로부터 응답을 받은 NAT장비는 NAT테이블에서 목적지 IP주소인 11.11.11.11이 원래10.10.10.10과 2000포트인것을 확인한다.
6. NAT장비는 NAT장비에서 확인한 목저지 IP주소와 포트 번호를 토대로 패킷을 재작성하고 사용자는 재작성된 패킷을 받아 웹페이지를 표시한다. 

- 출발지 10.10.10.10에서 20.20.20.20으로 보낼 때 NAT는 IP만 변경한다.
    
- 하지만 PAT는 IP뿐만 아니라 서비스 포트도 변경하기 때문에 하나의 IP만으로 다수의 사용자를 구분할 수 있다.
    
- 하지만 포트의 개수는 제한되어 있기 때문에 재사용된다.
    
- 서비스 포트가 모두 사용중이라면 정상적으로 사용할 수 없다.
    
- 만약 목적지 IP가 PAT IP라면 해당 IP가 어떤 IP에 바인딩 되는지 알 수 있는 NAT 테이블이 없기 때문에 통신할 수 없습니다.

## SNAT와 DNAT

NAT는 네트워크 주소를 변환할 때 어떤 IP 주소를 변경하느냐에 따라서 나뉜다.

- SNAT(Source) : 출발지 주소를 변경하는 NAT
    
- DNAT(Destination) : 도착지 주소를 변경하는 NAT

    
    어떤 주소를 변경할 지를 결정하는 것은 서비스 흐름과 목적에 따라 결정된다.
    
    SNAT와 DNAT의 기준은 NAT가 수행되기 전의 트래픽이 출발하는 시작 지점이다.
    
    즉 요청 시 SNAT를 해 목적지로 전송하면 해당 트래픽에 대한 응답을 받을 때는 출발지와 목적지가 반대로 되어 DNAT를 사용하게 되는데 이때 트래픽을 요청하는 시작 지점만 고려해 SNAT설정만 한다.
    
    NAT장비를 처음 통과할 때 NAT테이블이 생성되기 때문에 응답 패킷이 NAT에 들어오면 별도의 NAT설정이 없어도 NAT 테이블을 사용해 반대의 패킷을 변환해 줄 수 있다. 이 과정을 역 NAT라고 하며 NAT가 정상적으로 사용되기 위해서는 꼭 필요하다.
    
    <aside> 📄 표현의 다양성..
    
    ---
    
    실무에서는 NAT를 다양하게 표현한다고 한다.
    
    출발지 주소를 변경하는 NAT를 SNAT또는 Source NAT으로 표현하고 출발지 NAT라고도 표현한다.
    
    도착지 주소를 변경하는 NAT를 DNAT또는 Destination NAT로 표현하고 도착지 NAT라고도 표현한다.

## SNAT와 DNAT가 사용되는 경우

### Source NAT

- 사설에서 공인으로 통신할 때 많이 사용한다.
    
    공인 IP 주소의 목적지에서 출발지로 다시 응답을 받으려면 출발지 IP 주소가 필요한데 공인 대역에서는 사설 대역을 알 수 없다. 따라석 공인 IP를 목적지로 서비스를 호출할 때는 출발지를 별도의 공인 IP로 NAT해주어야 한다. 그래야 출발지 IP를 공인 IP로 확인해 응답할 수 있는 경로를 찾을 수 있다.
    
- 보안
    
    회사에서 외부와 통신할 때 내부 IP를 숨길 수 있다. 또한 회사의 내부의 IP와 외부 통신 기기의 IP와 겹쳐도 SNAT를 통해 IP가 변경하는데 사용될 수 있다.
    

### Destination NAT

- 로드 밸런서에서 많이 사용됨
    
    사용자는 서비스 요청을 할 때 로드밸런서에 설정된 서비스 VIP(Virtual IP)에 요청을 보내게 되면, 로드밸런서는 서버의 실제 IP로 DNAT해 내보낸다. ( 12장 로드밸런서 에서 더 자세히 다룬다고 함 )
    
- 대외망과의 네트워크 구성
    
    사내 IP는 중앙에서 관리하여 IP주소가 중복될 일이 적지만 대외망과 통신할 때에는 IP가 중복될 수 있다.
    
    IP주소가 중복되지 않더라도 IP주소가 제각각 이어서 신규 대외사와의 연동마다 라우팅을 설정해 주어야 한다. 이 경우 대외망 전용 NAT 대역으로 변경된 네트워크를 라우팅하면 되기 때문에 신규 대외사에 대해서 개별적으로 라우팅을 설정해줄 필요가 없고 사내 IP와 중복되는 IP가 있어도 라우팅 이슈 없이 구성할 수 있음
    
  
    

## 동적 NAT와 정적 NAT

- 정적 NAT : 출발지와 목적지의 IP를 미리 매핑해 고정해 놓은 NAT
    
- 동적 NAT : 출발지나 목적지가 어느 경우든 사전에 정해지지 않고 NAT를 수행할 때 IP를 동적으로 변경하는 NAT
