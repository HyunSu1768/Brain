### 물리 주소와 논리 주소

2계층에서는 어드레스로서 MAC주소를 사용, 하지만 3계층 에서는 MAC주소를 사용하지 않는다. → MAC주소에는 명확한 위치 정보가 없기 때문!

그래서 3계층을 위한 주소가 나왔다! → 논리 주소

- 물리 주소에서는…
    - 주소에 위치정보가 없기 때문에 수많은 컴퓨터 중에서 찾아내는것은 어렵다.
- 논리 주소에서는…
    - 위치정보가 포함되어 있어 손쉽게 찾아낼 수 있다.

## IP 주소 특징

- 계층형 주소이다.
    - A에 있다, A안에 있는 B에 있다. C라는 컴퓨터에 있다. 와 같이 세분화할 수 있다.
    - 대전광역시 유성구 가정북로 76
- 네트워크 관리자가 컴퓨터에 할당한다.
- 인터페이스가 고장이 나 인터페이스를 교체해도 논리 주소는 바뀌지 않는다.
- 소속된 네트워크가 바뀌면 IP 주소도 바뀐다.
## IP 주소 체계

- IPv4 와 IPv6로 나뉜다.
    - 두 버전 사이에는 호환성이 없다.

- ****************************32비트이다.****************************
    - 8개의 비트를 10진수로 표현하여 8비트 사이에 점을 찍는다 → 옥텟
    - 8비트를 Byte라고도 하지만, 네트워크에서는 옥텟을 사용한다.
    - 4개의 옥텟, 32비트로 네트워크 번호, 컴퓨터 번호를 나타낸다.


### 클래스

- 네트워크 주소 : 호스트들을 모은 네트워크를 지칭하는 주소
    - 네트워크 주소가 같은 네트워크를 로컬 네트워크라고 한다.
- 호스트 주소 : 네트워크 내 존재하는 호스트를 구분하기 위한 주소

네크워크 주소는 유일해야 하기 때문에 ICANN( The Internet Corporation for Assigned Names and Number) 라는 조직에서 네트워크 주소를 인터넷 사업자나, 기업에게 할당한다. → IP 주소를 유일하게 관리하기 위한 단체이다.

- 기업의 규모에 따라 IP의 범위를 변경하기 위해 클래스를 사용한다.
    
- A~E 클래스로 나뉘어 있다.




첫번째 옥텟의 맨 앞 몇 비트만 보면 된다.

0xxxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx → A클래스

- A 클래스의 범위
    - **0**0000000 ~ 01111111 : 0.0.0.0 ~ 127.255.255.255
        
        - A클래스에서 127번은 자기 자신을 나타내는 루프백 주소로 사용되어 실제 사용할 수 있는 주소는 1.0.0.0 ~ 126.255.255.255
        
        <aside> 💡 0.0.0.0 어디감?
        
        - 0.0.0.0은 현재 호스트 또는 모든 네트워크를 뜻하여 일반적으로 IP주소로 사용될 수 없다. </aside>

10xxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx → B클래스

- B 클래스의 범위
    - **10**000000 ~ 10111111 : 128.0.0.0 ~ 191.255.255.255

110xxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx → C클래스

- C클래스의 범위
    - **110**00000 ~ 11011111 : 192.0.0.0 ~ 223.255.255.255

1110xxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx → D클래스

1111xxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx → E클래스



## 클래스풀과 클래스리스

---

### 클래스풀 어드레싱

- 위에서 말한것과 같이 클래스 기반의 IP주소 체계를 클래스풀 이라고 한다.
- IP주소를 처음 만들었을 때에는 클래스 개념이 주소 낭비가 적고 최적의 조건을 만들 수 있었던 좋은 선택이었다.
- 네트워크 주소와 호스트 주소를 구분짓는 구분자인 서브넷 마스크가 필요가 없다.
- 또한 첫번째 옥탯의 일부만 보면 어떤 클래스인지 알 수 있다.

### 클래스리스 네트워크의 등장


A클래스는 1600만개

B클래스는 6만5천개

C클래스는 256개

각 클래스는 다음과 같은 IP 주소의 개수를 가진다.

자, 그럼 3000개 정도의 IP주소가 필요하다면 어떡할까?

그렇다. B클래스를 써야하고 6만5천개 중 3000천개 밖에 사용하지 못하니 낭비가 심하다.

인터넷은 계속 보급되어 가고있고

크기를 3개로만 구분하니 그 부분에 딱 맞아떨어지지 않으면 낭비가 많아졌다.

그래서 클래스리스 네트워크가 등장하였다!



### 클래스리스 네트워크

- 클래스라는 구분을 없앤 네트워크이다.

그렇다면 클래스리스 네트워크에서는 어디까지가 네트워크 주소이고, 어디까지 호스트 주소인지 구분할까?

→ 서브넷 마스크를 통해 구분한다.

### 서브넷 마스크

- 2진수 숫자 1은 네트워크 주소, 0은 호스트 주소를 나타낸다.



### 서브넷 마스크 표현 방법

- 2진수로 표현할 때는 1이 연속된 자릿수를 표현한다.
    - A클래스의 경우 첫번째 옥텟만이 네트워크 주소를 나타내기 때문에 /8 로 나타낸다.
    - B클래스 : /16
    - C클래스 : /24
- 10진수로 표현할 때
    - A클래스 : 255.0.0.0
    - B클래스 : 255.255.0.0
    - C클래스 : 255.255.255.0

## 서브네팅

- 원래 부여된 클래스의 기준을 무시하고 새로운 네트워크-호스트 구분 기준을 정해 기존 클래스풀 단위의 네트워크보다 더 쪼개서 사용하는것을 말한다.
- 네트워크 사용자의 서브네팅

## 공인 IP와 사설 IP

- 인터넷에 접속하기 위해서는 공인 IP가 필요하다.
- 하지만 인터넷에 접속하지 않아도 되고, 개인적으로 네트워크를 구성한다면 사설IP를 써서 공인 IP를 할당받지 않고도 네트워크를 구축할 수 있다.
- 또한 NAT를 사용하여 사설 IP를 공인 IP로 바꾸어 사설 IP를 사용할 수도 있다.
